# 硅谷范式 · Silicon Valley Paradigms /ˈpærədaɪmz/

## 构建原则

| 原则 | 英文 | 含义 |
|------|------|------|
| 薄壳厚核 | Thin Shell, Thick Core | 入口层薄，引擎层厚 |
| 零依赖 | Zero Dependency /ˈziːroʊ dɪˈpɛndənsi/ | 标准库优先 |
| 声明式 | Declarative /dɪˈklærətɪv/ | 用户说意图，系统做推理 |
| 流式反馈 | Streaming Feedback | 过程可见，不黑盒 |
| 错误当产品 | Error as Product | 出错时用户不慌 |

## 工程规范

```
PRD (1页) → API 契约 → 写代码 → Harness → Red Flags → 发布
```

- Harness /ˈhɑːrnɪs/: 每函数 ≥ 1 测试，改前改后都跑
- Red Flags: 发布前全 ✅（不横向滚动、无框线字符、引擎直出不经模型）

## 决策框架 — ADR

Architecture Decision Record /ˈɑːrkɪtektʃər dɪˈsɪʒən ˈrekɔːrd/

```
标题: 简短动词短语
状态: proposed → accepted → deprecated → superseded
上下文: 为什么需要决策
决策: 做了什么选择
后果: 正面 + 负面
```
