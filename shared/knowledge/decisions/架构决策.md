# 架构决策 · ADR /ˈɑːrkɪtektʃər dɪˈsɪʒən ˈrekɔːrd/

## 2026-07-04: FIPA v0.2 消息协议

- **状态**: accepted /əkˈsɛptɪd/
- **上下文**: Claude↔星辰 inbox/outbox 通道已跑通，需要统一多 Agent 通信
- **决策**: 采用 FIPA 风格消息路由（非黑板），统一 `{from, to, type, payload, status}`
- **后果**: 
  - ✅ 三 Agent 同一套协议，加第四个无需改代码
  - ⚠️ Claude Code 非 server 模式，消息需人工触发 SessionStart 扫描

## 2026-07-04: 文件通道选 A

- **状态**: accepted
- **上下文**: Claude Code 是交互式 CLI，没有 MCP server 模式。文件落盘+轮询是松耦合异步通信
- **决策**: 接受现状。文件通道撑到第一个真实客户上线再考虑升级
- **升级条件**: 日活摄影师 > 20，日匹配 > 10，有付费客户
- **后果**: 
  - ✅ 零新代码，稳定
  - ⚠️ 延迟 1-几分钟（异步任务本身耗时同级，可接受）

## 2026-07-04: Phase 1-5 只做 Phase 0

- **后果**:

## 2026-07-04: 四 Agent 闭环架构

- **状态**: accepted
- **上下文**: 基于 AIMA §2.4.7 学习型 agent 模板，每个 Agent 内部自成闭环
