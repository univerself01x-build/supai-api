# Software 2.0

**软件 2.0** · Software 2.0 · /ˈsɒftwɛə tuː pɔɪnt əʊ/

Andrej Karpathy 在 2017 年提出的范式转移理论。核心论断："数据集是新的源代码。"

## 三个范式

| 范式 | 编程方式 | 特征 |
|------|---------|------|
| **Software 1.0** | 手写代码（Python, C++） | 精确逻辑，人类可读 |
| **Software 2.0** | 神经网络权重（数据+训练） | 近似逻辑，"代码"由优化器搜索 |
| **Software 3.0** | 自然语言 + 模型编排 | "最热门的编程语言是英语"（Karpathy 2023） |

## 这意味着什么技能在变化

**正在贬值：**
- 手写复杂算法
- 深度学习数学推导（AI Engineer 不需要）
- PyTorch 底层编程

**正在升值：**
- Context Engineering
- Evals-Driven Development
- Agentic System Design
- AI-Native Product Thinking

## 为什么这对 AI Agent 工程师重要

峰哥当前的开发范式（Python 标准库 engine + LLM API 编排）就是 Software 3.0 的实践。理解这个范式转移，就知道为什么"不学 PyTorch 底层"不是偷懒，是正确定位。
