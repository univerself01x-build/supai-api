# Tutorial Hell

**教程地狱** · Tutorial Hell · /tjuːˈtɔːriəl hɛl/

反模式：学习者陷入不断观看/跟随教程但不独立构建项目的循环，始终无法脱离"脚手架"独立编程。

## 典型症状

1. 完成 100 门短课程但 0 个独立项目
2. 看完视频觉得"懂了"，打开空白文件不知道该写什么
3. 每个新框架都要学一遍，但从未深入任何一个
4. 学习进度按"刷了多少课"衡量而非"构建了什么"
5. 遇到问题第一反应是"找教程"而非"读文档/看源码"

## 为什么这对 AI Agent 工程师重要

峰哥已有的免疫策略：**实战驱动**（速派）→ Phase 复盘 → 知识卡片。每门课必须对应当前项目的具体需求。学完立即用在速派上。

## 预防策略

1. **1:1 规则** — 每学 1 小时课，至少花 1 小时独立写代码
2. **Build in Public** — swyx 推荐公开构建过程来获取反馈
3. **按需学习** — 只学当前项目需要的，不囤积
4. **先做后学** — Chip Huyen 式：先动手，遇到问题再查
