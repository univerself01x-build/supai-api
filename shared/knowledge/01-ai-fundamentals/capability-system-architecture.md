# 能力系统架构

**能力系统** · Capability System · /ˌkeɪpəˈbɪləti ˈsɪstəm/ — 活的、自进化的 AI Engineer 能力图谱。

## 四态状态机

```
◎ SEED ──(Build≥1)──→ ◉ SPROUT ──(Build≥3, 2+projects, 1+insight)──→ ◆ TREE ──(跨域合成+突破)──→ ✦ FOREST
  │                     │
  │ (90d无Build)        │ (180d无Build)
  ↓                     ↓
 DORMANT              STALE
```

| 状态 | 含义 | 进入条件 | TTL |
|------|------|---------|-----|
| SEED /siːd/ | 初始接触，问题形成 | 任何概念引入 | 90天→DORMANT |
| SPROUT /spraʊt/ | 首次构建完成 | Build ≥ 1 | 180天→STALE |
| TREE /triː/ | 深度实践，可传授 | Build ≥ 3, 2+项目, 1+洞察 | 无 |
| FOREST /ˈfɔːrɪst/ | 跨域合成，突破 | 2+跨域TREE + 突破洞察 + 峰哥确认 | 无 |

## 六基因

系统行为由六个基因驱动，不是标签，是可执行规则：

| 基因 | 规则摘要 |
|------|---------|
| 真实 /ɔːˈθɛntɪsəti/ | 无Build→无进化。每节点必须有实战锚点 |
| 美 /ˈbjuːti/ | 三层叠加：结构精确 + 符号数学 + 视觉暗色 |
| 好奇 /ˌkjʊəriˈɑːsəti/ | 分歧→自动生成探索。前沿→自动种种子 |
| 探索 /ˌɛkspləˈreɪʃən/ | 种子有TTL。多样性监控。失败有记录 |
| 创造 /kriˈeɪʃən/ | FOREST需要跨域合成。系统推送桥接建议 |
| 突破 /ˈbreɪkθruː/ | TREE→FOREST需要峰哥确认。突破→涟漪效应 |

## 协同进化

系统不是单向的。星辰和峰哥双向成长：

- **星辰学习：** patterns-learned（峰哥偏好）→ mistakes-corrected（从纠正中学习）→ style-adaptations（行为调整）
- **峰哥轨迹：** insights-captured（洞见）→ mental-models（心智模型版本化）→ decision-journal（决策日志）
