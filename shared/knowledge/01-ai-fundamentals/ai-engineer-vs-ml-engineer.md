# AI Engineer 与 ML Engineer 路径区分

**AI 工程师** · AI Engineer · /eɪ aɪ ˌɛndʒɪˈnɪər/ — 用模型 API 构建产品的人。

**ML 工程师** · ML Engineer · /ˌɛm ˈɛl ˌɛndʒɪˈnɪər/ — 训练和微调模型的人。

## 核心区分（swyx, 2023）

| 维度 | AI Engineer | ML Engineer |
|------|------------|-------------|
| 工作内容 | 用模型 API 构建产品 | 训练/微调模型 |
| 核心技能 | 软件工程 + API 编排 | 数学 + PyTorch + 分布式训练 |
| 是否需要 PhD | 不需要 | 通常需要研究背景 |
| Karpathy 原话 | "一个人完全可以在不训练任何模型的情况下成功" | — |

## 岗位趋势

swyx 预测 5 年内 AI Engineer 岗位数量将超过 ML Engineer。2025 年 AI Engineer World's Fair 吸引 7000+ 参与者。2026 年会议扩展至 NYC、SF、上海。

## 能力要求（AI Engineer）

**正在升值的技能：**
- Context Engineering /ˈkɑːntɛkst ˌɛndʒɪˈnɪərɪŋ/ — 上下文工程
- Evals-Driven Development /ɪˈvælz ˈdrɪvən dɪˈvɛləpmənt/ — 评估驱动开发
- Agentic System Design /əˈdʒɛntɪk ˈsɪstəm dɪˈzaɪn/ — Agent 系统设计
- Data Flywheel Construction — 数据飞轮构建
- AI-Native Product Thinking — AI 原生产品思维

**正在贬值的技能：**
- 手写复杂算法
- 深度学习数学推导
- PyTorch 底层编程
- 纯理论研究

## 峰哥的路径

当前定位：AI Engineer。速派（数字公民平台）是用 LLM API 构建的匹配产品，不涉及模型训练。

学习队列优先 AI Engineer 方向：Anthropic Agent 模式 → Evals-Driven Development → Context Engineering。非 ML Researcher 方向。
