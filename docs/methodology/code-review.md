# 速派 · Code Review 流程 v1.0

> 2026 共识：45% AI 生成代码含漏洞。审查不是可选项。
> 参考：Vibe Coding Hangover 报告、"不审不合并"原则

## 触发条件

| 情况 | 审查者 |
|------|--------|
| Claude 提交代码 | 星辰审 |
| 星辰提交代码 | Claude 审 |
| CodeX 生成代码 | Claude 审（品味）+ 星辰审（功能） |
| 架构变更 | Claude + 星辰 + 峰哥确认 |

## 审查维度

| 维度 | 检查什么 |
|------|---------|
| **正确性** | harness 全绿？逻辑对？ |
| **边界** | 空输入？异常输入？超时？ |
| **品味** | 命名？结构？有没有更简洁的写法？ |
| **范式** | 是否符合 Agent-Native？SSOT？薄壳厚核？ |
| **安全** | SQL 注入？key 泄露？权限绕过？ |

## 流程

```
提交者 → 写 inbox 通知审查者 + 链接到 diff
审查者 → 看代码 → 三条以内反馈 → Accept / Request Changes
Accept → 合并
Changes → 提交者修 → 再审查
```

## 审查规则

1. 每条反馈必须具体到行。不说"可以更简洁"
2. 审查者 4h 内响应（非阻塞可放宽）
3. 紧急修复可先合并后审
4. CodeX 代码必须双审（品味+功能）
