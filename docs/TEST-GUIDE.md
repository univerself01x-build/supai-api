# 速派 · 测试指南 v1.0

> 每次改完引擎或前端，跑一遍。客户演示也用这份。

---

## 一、输入格式

**一句话模板：** `[时间] [地点] 有场 [活动类型]，预算 [金额]，需要 [角色]，要求 [条件]`

**客户自然语言也接受。** 不用强制格式——系统能从自然语言提取关键信息。

### 关键词速查

| 客户说 | 系统理解 |
|--------|---------|
| 下周/这周末/明天下午 | 时间（Phase 1 不做档期校验，仅展示） |
| 上海/北京/深圳 | 地点（同城优先 +15 分） |
| 发布会/AI峰会/Demo Day | 首发定制档（enterprise） |
| 论坛/沙龙/年会 | 资深档（premier） |
| 团建/探店/门店/活动 | 专业档（express） |
| 写真/跟拍/证件照 | 速拍档（pool） |
| 拍得好一点 | 系统取评分最高的 |
| 会拍短视频 | 匹配有短视频技能的摄影师 |
| 预算 5000/8000 | 影响档位判定 |

### 不需要填的
- 姓名、手机号（无需注册）
- 摄影师指定（系统自动匹配）
- 设备要求（Phase 2）

## 二、测试用例

### 四档覆盖

**首发定制档**
```
下周五上海AI产品发布会，预算8000，需要摄影师和剪辑师
```
期望：tier=enterprise，推荐 enterprise 级摄影师（4.8★+/30+单），佣金 25%

**资深档**
```
下周北京有个技术沙龙，预算5000，要一个摄影师，拍得好一点的
```
期望：tier=premier，推荐 premier 级摄影师（4.5★+/15+单），佣金 20%

**专业档**
```
这周末上海团建，预算1500，找个摄影师跟拍
```
期望：tier=express，推荐 express 级摄影师（4.0★+/5+单），佣金 15%

**速拍档**
```
帮我找个人拍证件照，预算200
```
期望：tier=pool，任何档位摄影师，佣金 10%

### 边界场景

**预算极低**
```
随手拍一下，预算50
```
期望：正常匹配，tier=pool

**无地点**
```
找个摄影师拍产品
```
期望：默认上海，所有城市摄影师参与

**模糊需求**
```
需要拍照的
```
期望：匹配到活动摄影/产品拍摄，不报错不空白

**英文混输**
```
Shanghai AI Conference, budget 8000
```
期望：匹配到活动摄影，档位正确

## 三、验收清单

### API 层（后端）

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | `POST /api/match` 返回 projectId | 每次新请求生成唯一 ID |
| 2 | tier 字段正确 | 匹配 TIER_EVENTS 规则 |
| 3 | recommendedTeam 非空 | 至少 1 个摄影师 |
| 4 | alternativeOptions 有备选 | 2-3 个备选 |
| 5 | summary 可读 | 一句话描述匹配结果 |
| 6 | `GET /api/projects` 返回列表 | 包含所有已创建项目 |
| 7 | `GET /api/projects/:id` 返回详情 | 含 Team/时间线 |
| 8 | `POST /:id/confirm` 更新状态 | 确认后 status 变为 confirmed |
| 9 | harness 24/24 | 全绿 |

### 前端层（React）

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 输入框可见 | 页面打开即可见 |
| 2 | 输入需求后 2 秒内出现新卡片 | 匹配响应及时 |
| 3 | 卡片显示：标题 + 档位标签 + 城市 + 预算 + 摄影师 | 五要素齐全 |
| 4 | 档位标签颜色区分 | 首发定制/资深/专业/速拍 不同视觉 |
| 5 | 点击"复制" → 粘贴到微信 | 文本格式完整，确认链接可打开 |
| 6 | 手机打开确认链接 | 显示项目详情 + 确认/拒绝按钮 |
| 7 | 点"确认接单" | 显示成功提示 |
| 8 | 点"暂不接单" | 显示已拒绝提示 |
| 9 | 暗色主题 | 背景深色，文字可读 |
| 10 | Console 零错误 | 无 hydration error / CORS error |

### 端到端

| # | 检查项 | 标准 |
|---|--------|------|
| 1 | 输入 → 匹配 → 复制 → 发群 → 确认 | 全链路无中断 |
| 2 | 手机端确认页可用 | 微信内打开，按钮可点 |
| 3 | 多个需求连续输入 | 每次都产生新卡片，不互相覆盖 |

## 四、已知限制

- 不做真实档期校验（时间是文本展示）
- 不做摄影师反选（确认页给第一位摄影师）
- 不做多角色匹配（团队只有 photographer slot）
- 不做客户注册（无登录态）
